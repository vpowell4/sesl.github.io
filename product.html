SESL Product Overview

SESL is a deterministic, rule-based expert system language that turns structured content into reliable, traceable decision logic.

- Deterministic

SESL uses a forward-chaining rules engine that produces the same result every time â€” no randomness, no hallucinations, no inconsistencies. SESL integrates with most AI platforms to convert documents or text into SESL rules. You can run SESL interactively using your favorite LLM or locally using the SESL CLI.

- Explainable

SESL provides rule-level trace output showing exactly how a result was reached. This makes it ideal for compliance and audit-driven environments.

- Readable and Extensible

SESL uses a human-friendly language syntax designed for clarity and ease of understanding. It can be applied to any domain â€” from business processes and legal reasoning to diagnostics across industries and academic fields. SESL scales to thousands of rules and has been tested across a wide range of real-world use cases.

SESL Language

Structure of a SESL program

SESL programs define rules and facts. Rules are grouped in rulesets and facts are grouped in Scenarios. A SESL program can also include comments and metadata. Comments help to explain the rules and facts, and meta data can be used to provide additional information about the program. A typical SESL program might look like this: 

  
const:
  MIN_ADULT_AGE: 16

rules:

  - rule: check_age
    priority: 10
    if:
      - applicant.age < MIN_ADULT_AGE
    then:
      result.eligible: false
    because: "Applicant is under the required adult age."

  - rule: check_nationality
    priority: 20
    if:
      - applicant.nationality != "British"
    then:
      result.eligible: false
    because: "Applicant is not a British national."

  - rule: check_previous_passport
    priority: 30
    if:
      - applicant.has_previous_passport == true
    then:
      result.eligible: false
    because: "Applicant has already had a UK passport."

  - rule: verify_documents
    priority: 40
    if:
      all:
        - docs.birth_certificate == true
        - docs.identity_evidence == true
        - docs.photo_ok == true
    then:
      facts.docs_ok: true
    because: "All required documents are provided."

  - rule: final_decision
    priority: 100
    if:
      - facts.docs_ok == true
      - result.eligible != false
    then:
      result.eligible: true
    because: "All requirements satisfied."

facts:
  - name: Scenario1
    applicant:
      age: 22
      nationality: "British"
      has_previous_passport: false
    docs:
      birth_certificate: true
      identity_evidence: true
      photo_ok: true

How SESL processes the rules and facts

When you run a SESL program either within a LLM tool or locally using the SESL CLI, the SESL engine processes the rules and facts to derive conclusions. It starts with the initial facts defined in the scenarios and applies the rules in the rulesets. SESL works through these rules updating the facts until no more rules can be applied and a stable state is reached. This provides a consistent and repeatable answer.

The final output is a set of derived facts that represent the conclusions reached by the SESL engine or if it is not possible to reach a conclusion SESL will indicate that as well.

SESL CLI

The SESL CLI lets you create, lint, explain, and run rule files locally. 

# Run a SESL file
sesl rules.sesl

# Run with explanation
sesl describe rules.sesl

# Lint your rules
sesl lint rules.sesl

# Run a scenario
sesl run rules.sesl --scenario Example
