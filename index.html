What is SESL?

SESL (Simple Expert System Language) is a **forward-chaining expert-system engine** with a clean, human-readable syntax. It converts text or documents into **rules**, **facts**, and **deterministic logic paths**.

Every SESL answer comes with an **explicit explanation** — which rules were triggered, what facts were used, and the “because” behind each step. SESL works inside ChatGPT or completely independently on a local machine or server.

  • Readable by non-programmers
  • Deterministic & reproducible
  • Traceable & audit-friendly
  • Safe under malformed input

Why SESL exists

LLMs are powerful, but fundamentally **probabilistic**. The same question may produce different answers. They can generate text that looks correct but is factually wrong — a hallucination.

SESL solves this by replacing probability-based reasoning with **explicit rules and facts**. The output is always:

  • Predictable
  • Consistent
  • Auditable
  • Scientifically traceable

Example Use Cases

SESL is ideal wherever decisions must be **consistent**, **auditable**, and **rule-driven** — such as government processes, compliance checks, risk evaluations, pricing engines, or any expert-system logic.

Eligibility logic: passport applications, benefits checks, renewals.
Compliance: policy rules, constraints, validations, audit requirements.
Risk & scoring: thresholds, weighted scoring, decision trees.
Pricing engines: deterministic price calculations and adjustments.

SESL Command Line Interface

SESL includes a full-featured CLI that allows you to build, run, lint, inspect, and debug SESL models outside of ChatGPT.

  • Load & save SESL files
  • Inspect rules, facts, scenarios
  • Run scenarios normally or with explanation
  • Inline or multiline rule editing
  • Full linting with error & warning reporting
  • Dependency graph generation
# Run a SESL file
sesl rules.sesl

# Run with explanation
sesl describe rules.sesl

# Lint your rules
sesl lint rules.sesl

# Run a specific scenario
sesl run rules.sesl --scenario Case1
