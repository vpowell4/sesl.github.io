MCP & API Usage Policy

This document explains how SESL manages usage, billing, performance, and data handling for MCP and API services.

1. Billing & Minimum Commitments

  • MCP requires a minimum monthly spend of £10.
  • API usage requires a minimum monthly spend of £10.
  • A valid credit card is required for both services.
  • Organisation Server customers are billed annually with a 12-month minimum commitment.

2. Rate Limits & Fair Use

  • Requests are processed on a best-effort basis under shared plans.
  • High-volume workloads may be rate-limited if they exceed fair-use thresholds.
  • Enterprise plans may reserve dedicated capacity.

3. Data Processing

  • SESL does not retain rule files or scenario data submitted via MCP or API unless logging is explicitly enabled.
  • We only store anonymised metadata needed for billing and service diagnostics.
  • No model training uses customer data.

4. Region Selection

MCP and API servers operate in the US, EU, and UK. Enterprise customers may choose a dedicated region or request private hosting.

5. Security

  • All traffic is encrypted in transit (TLS 1.2+).
  • All data is encrypted at rest.
  • Enterprise options include isolated compute, private networking, and optional on-prem deployment.

6. Service Availability & SLAs

  • Shared plans operate on a best-effort uptime basis.
  • Enterprise plans include custom SLAs covering uptime, latency, and response times.

7. Termination

  • Customers may cancel MCP or API service after meeting the minimum monthly commitment.
  • Enterprise service requires 90-days notice after the 12-month commitment period.
